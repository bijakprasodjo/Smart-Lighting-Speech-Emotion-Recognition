Data and Computer Program Availability Evidence Inventory
============================================================

Repository:
https://github.com/bijakprasodjo/Smart-Lighting-Speech-Emotion-Recognition

Paper:
Lightweight Multi-Branch Dilated 1D CNN for Edge-Based Speech Emotion Recognition with Adaptive Smart Lighting Feedback via MQTT

Purpose:
This inventory maps the paper's major data, model, edge deployment, MQTT, and smart-light actuation claims to the files available in the repository.

1. Dataset and Preprocessing Evidence
-------------------------------------

1.1 Dataset preparation
File:
notebooks/01_dataset_preparation.ipynb

Supports:
- Multi-corpus dataset construction using RAVDESS, EMODB, TESS, and SAVEE.
- Label harmonization into four classes: angry, happy, neutral, and sad.
- Total utterance count: 2,911.
- Class counts: angry = 779, happy = 723, neutral = 695, sad = 714.
- Stratified train/validation/test split: 2,328 / 291 / 292 files.

Notes:
- Raw third-party datasets are not redistributed in this repository.
- Users must download RAVDESS, EMODB, TESS, and SAVEE from their original sources.

1.2 Feature extraction and augmentation
File:
notebooks/02_preprocessing_features.ipynb

Supports:
- Audio resampling to 16 kHz.
- Fixed duration of 3 seconds.
- MFCC extraction with n_mfcc = 40, n_fft = 512, hop_length = 256.
- Feature shape: (188, 40).
- Per-dataset StandardScaler normalization.
- Training-only data augmentation:
  - Gaussian noise injection with noise ratio 0.005.
  - Time stretching with rate range 0.85 to 1.15.
  - Pitch shifting from -3 to +3 semitones.
- Training expansion to 9,312 samples.

Related runtime file:
src/preprocessing.py

Related scaler files:
models/scalers/scaler_ravdess.pkl
models/scalers/scaler_emodb.pkl
models/scalers/scaler_savee.pkl
models/scalers/scaler_tess.pkl

4. Raspberry Pi Runtime Code Evidence
-------------------------------------

4.1 Audio stream
File:
src/audio_stream.py

Supports:
- Continuous microphone capture.
- Native microphone sampling rate: 44.1 kHz.
- Resampling to 16 kHz.
- Stereo-to-mono conversion.
- 3-second circular buffer for sliding-window inference.

4.2 Runtime preprocessing
File:
src/preprocessing.py

Supports:
- Runtime audio preprocessing.
- Silence trimming, RMS normalization, noise-floor suppression, padding/cropping.
- MFCC extraction with shape (188, 40).
- Per-dataset scaler loading from models/scalers.

4.3 TFLite INT8 inference
File:
src/inference.py

Supports:
- Loading the INT8 TensorFlow Lite model.
- Input tensor shape: (1, 188, 40).
- Emotion label mapping:
  - 0 = angry
  - 1 = happy
  - 2 = neutral
  - 3 = sad
- Probability output and inference latency measurement.

4.4 Streaming pipeline
File:
src/main_stream.py

Supports:
- 3-second inference window.
- 0.5-second slide interval.
- 83.3% effective overlap.
- Confidence thresholding.
- Spectral flatness speech-like filtering.
- Majority voting using a 6-frame rolling buffer.
- Stable prediction publication through MQTT.
- Sonoff local HTTP actuation.
- JSONL runtime logging.

4.5 MQTT publication
File:
src/mqtt_client.py

Supports:
- Local Mosquitto broker connection.
- MQTT QoS 1 publication.
- Topics:
  - ser/emotion
  - ser/lighting
  - ser/status

4.6 Sonoff smart-light actuation
File:
src/sonoff_controller.py

Supports:
- Sonoff B05 local HTTP API actuation.
- Emotion-to-lighting mapping:
  - angry: cool blue, low brightness
  - happy: neutral white, high brightness
  - neutral: warm white, medium brightness
  - sad: warm yellow, high brightness
- Gradual transition for same-mode lighting changes.
- Direct transition for cross-mode color/white changes due to API constraints.

7. Files Not Included
---------------------

The following are intentionally not included:

- Raw RAVDESS, EMODB, TESS, and SAVEE audio files.
  Reason: third-party licensing terms.

- Generated feature arrays such as .npy files.
  Reason: derived directly from third-party audio datasets and unnecessary if the user runs the preprocessing notebook.

- Local environment files, Wi-Fi credentials, real device IDs, and API keys.
  Reason: security and privacy.
